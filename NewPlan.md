│ PART 1 — BRUTAL ROAST (current state)                                                                           │
│                                                                                                                 │
│ Overall: 6.8 / 10. Real engineering underneath — verified-identity auth, fail-closed rate limiting, a genuine RERA-hallucination gate, SSE streaming with fallback discipline, DB-backed chip inventory, thorough frontend error states and code-splitting. This is well above "weekend project."                                          │
│                                                                                                                 │
│ But it is not production-safe today, and several defects are demo-breaking:                                     │
│                                                                                                                 │
│ - The admin intelligence layer is silently dead. Half-finished auth migration: IntelligenceWorkspace, InvestmentInsightsEditor, LocationIntelligenceEditor, ImageUpload all call the API with credentials:'include' and no Bearer token — every save returns 401 and swallows the error. You cannot populate the AI intelligence data the whole product depends on. And when you fix auth, the editors wipe each other's data (peer-overwrite of one shared intelligence_data JSON column against a stale page-load snapshot).                                   │
│ - Chips repeat. Clarifying-chip IDs embed Date.now() (conversationEngine.ts:234,247,261) so ID dedup can never match them; DB dedup only keeps the last 50 IDs. The one feature the owner cares most about (progressive, non-repeating, predictive chips) is the buggiest.                                                               │
│ - The detail panel's Pricing/Residences tabs are broken. Payment-plan and cost-sheet fetch gates check a tab name ('Floor Plans & Pricing') that was renamed and no longer exists — so those tabs always render null (ProjectDetailPanel.tsx:180,190).                                                                               │
│ - Fabricated data renders as real. 'G+26' floors fallback (PropertyDetailPanel.tsx:571), a hardcoded Sector-150 price chart (PriceTimeline.tsx), a hardcoded Sector-150 heatmap (SectorHeatmap.tsx), and a 'Sector 10'/'Greater Noida West' market fallback (IntelligenceTab.tsx:638). This directly violates the "never invent" product rule.                                                                                                   │
│ - Hallucination gate is half-built. RERA numbers are validated against the DB (excellent), but prices, project names, and BHK are not — a non-compliant model can emit "4BHK in Sector 150 for ₹50L" and pass every guardrail. │
│ - Backend is blind in production. Frontend has proper Sentry; the Express service (chat, guardrails, AI calls, leads, admin) has zero exception aggregation — only stdout.                                                     │
│ - Deploy config is ambiguous. Two vercel.json files with different settings; the chat route's maxDuration:60 likely isn't applied (10s timeout risk).                                                                        │
│ - Safety code is untested. guardrails.ts, the chat route, and the conversation/scoring engine have no tests — the exact code that must not regress.                                                                           │
│ - Indirect prompt injection is open. Fetched web/RERA page content is handed to the model unwrapped (chat.ts:690, web.ts:157) — a poisoned page can inject instructions.                                            │
│ - Lint masks real bugs. .eslintrc.json:10-12 turns off no-explicit-any, no-unused-vars, and react-hooks/exhaustive-deps globally — hiding genuine missing-dependency hook bugs.                             │
│                                                                                                                 │
